Dropdown is a single-select combobox: a labelled field that opens a menu of options and shows the chosen one. It supports a label, helper text, required/optional marking, validation states, and a scrolling menu for long option lists.

## When to use it

Use a dropdown to let a user pick **one** option from a list that's too long to show comfortably inline, or where vertical space is tight. As a rough guide, reach for it once a list passes five to seven options.

Reach for a different control when:

- **The list is short (under ~5)** — radio buttons show all options at once with no extra click, which is usually better.
- **Users can pick more than one** — this is single-select only. Use a multi-select pattern or a list of checkboxes.
- **The choice is binary** — use a checkbox or toggle, not a two-item dropdown.
- **The items are actions, not values** — use a menu button. A dropdown selects a value; it doesn't trigger commands.

## Label, required and optional

Always give a dropdown a visible label. Bare placeholder-as-label fails the moment a value is selected and the placeholder disappears.

Required and optional marking is mutually exclusive — a field is required (shows `*`), optional (shows "(optional)"), or neither. Pick one convention per form and hold to it: mark every optional field, or mark every required field, but don't mix both in the same form or you double the user's cognitive load.

## Placeholder vs value

The field shows the selected value if there is one, otherwise the placeholder, otherwise nothing. Write the placeholder as an instruction ("Select a country"), not as a fake option. Don't rely on it to carry meaning the label should hold.

## Validation

| State | Use it for |
| --- | --- |
| **Default** | The normal resting state. |
| **Error** | A failed validation. Pair it with `errorText` in the message row — the error styling alone gives no explanation. |
| **Success** | An optional confirmation treatment for a valid selection. Use sparingly; constant green ticks become noise. |

## Accessibility essentials

The detail lives in the developer spec below; these are the points a designer needs to get right.

<AccordionGroup>
  <Accordion title="Two focus stops: the field, then the options" icon="list">
    The field is always stop one. When the menu opens, each option becomes reachable. The label, value, helper text, required/optional marker all merge into the field's announcement — none is a separate stop.
  </Accordion>

  <Accordion title="A real select is announced as a combobox" icon="square-check">
    Screen readers announce it as a combobox/drop-down with its expanded/collapsed state. This comes for free with a native control; custom implementations have to recreate it carefully (see the Voice tab).
  </Accordion>

  <Accordion title="The error message attaches to the field" icon="triangle-exclamation">
    Don't announce the error icon as its own element. The invalid state and message bind to the field (`aria-invalid` \+ the described-by error text), read as part of stop one.
  </Accordion>

  <Accordion title="Chevron and scrollbar are decorative" icon="eye-slash">
    The chevron and the menu scrollbar carry no meaning for assistive tech and must be hidden from it.
  </Accordion>

  <Accordion title="Disabled removes the field from focus order" icon="ban">
    A disabled dropdown can't be focused or opened. Give context for why it's unavailable rather than leaving the user guessing.
  </Accordion>
</AccordionGroup>

  <Tab title="API">
    > The Field resolves its displayed content by priority: selected `value`, else `placeholder`, else empty. The Figma `Content` axis (Empty / Selected Option / Placeholder Text) is the visual projection of that state, not a runtime prop. `requirement` is mutually exclusive (required / optional / none), merged from two Figma booleans. `isMenuOpen` drives `aria-expanded`; `hasScrollingMenu` only matters when the menu is open.

    ### Properties

    | Property | Type | Values | Default | Notes |
    | --- | --- | --- | --- | --- |
    | validationState | variant | default \| success \| error | default | Validation styling and messaging. error pairs with errorText; success is a confirmation treatment. |
    | isDisabled | boolean | true \| false | false | Persistent disabled state (Hover/Focus are transient, not props). Exposes `aria-disabled=true`. |
    | isMenuOpen | boolean | true \| false | false | Whether the options menu overlay is shown. Drives `aria-expanded`. |
    | value | text | (string) | – | Selected option label shown in the Field. Renders the "Selected Option" content state. |
    | placeholder | text | (string) | – | Shown when no value is selected. Renders the "Placeholder Text" content state. |
    | hasLabel | boolean | true \| false | true | Whether the Label container renders. |
    | label | text | (string) | "Label" | Field label. Only visible when `hasLabel=true`. |
    | requirement | variant | none \| required \| optional | none | Mutually-exclusive marker. required exposes `aria-required=true`. |
    | hasHelperText | boolean | true \| false | true | Whether the Helper text row renders below the Field. |
    | helperText | text | (string) | "Helper text" | Supporting text. Only visible when `hasHelperText=true`. |
    | hasErrorText | boolean | true \| false | true | Whether the Error text row renders. Typically paired with `validationState=error`. |
    | errorText | text | (string) | "Error text" | Validation message. Only visible when `hasErrorText=true`. |
    | hasScrollingMenu | boolean | true \| false | true | Whether the open menu shows a scroll bar. Only meaningful when `isMenuOpen=true`. |

  <Tab title="Structure">
    > Dropdown has no size or density axis. Validation and State are predominantly colour/visual; the only structural deltas are the field border weight (thickens on Hover/Focus) and the Error state, which inserts an error-text row and a 4px left accent border. Content swaps which text child renders in the Field with no dimensional change. Host container is 320 wide in the design source and owns no padding; each zone owns its spacing. Field insets use fluid spacing tokens; structural gaps use fixed tokens.

    ### Typography

    | Element | Family | Weight | Size | Line height | Style |
    | --- | --- | --- | --- | --- | --- |
    | label | Gotham | Bold | 16 | 23 | Body/M Compact Bold |
    | value / placeholder | Gotham | Book | 16 | 23 | Body/M Compact |
    | helperText / errorText | Gotham | Book | 14 | 22 | Body/S |
    | optionalTagLabel | Gotham | Book | 16 | 23 | Body/M Compact |

    ### Composition

    Four zones stack vertically: Label container, Field trigger, Menu overlay (open only), Helper text container. In the Error state an error-text container and left accent border are inserted between field and helper.

    | Zone | Layout | Notes |
    | --- | --- | --- |
    | Host container | vertical, hug | 320 wide in design, expected to fill in code; itemSpacing 0 |
    | Label container | horizontal, fill / hug | Label text \+ requirement marker; paddingBottom `spacing/fixed/xxs` (8) |
    | Field | fill / hug | Trigger row, minHeight 44; value/placeholder \+ chevron |
    | Menu container | fixed / fixed | Overlay; collapses to 0 height when closed |
    | Helper text container | fill / hug | Assistive line; paddingTop `spacing/fixed/xxs` (8) |

  <Tab title="Voice / Screen reader">
    > Dropdown is a single-select combobox: the Field trigger opens a popup listbox. Label, requirement marker, selected value/placeholder, and helper text all merge into the Field's announcement — not separate stops. Two focus stops: (1) the Field, always present; (2) each Menu item, reachable only while open. On web the listbox uses `aria-activedescendant` so DOM focus stays on the combobox; iOS and Android move focus into the option rows. Error text is the Field's error description, not a focus stop. Disabled removes the Field from focus order entirely.

    ### Focus order

    | # | Part | Announcement | Role | Notes |
    | --- | --- | --- | --- | --- |
    | 1 | Field | "Label, \[value or placeholder\], combo box, Helper text" | combobox / popup button | Has popup: listbox |
    | 2 | Menu item | "Option, X of N" | option (within listbox) | Menu open only |

    ### Closed — ARIA

    | Property | Value | Notes |
    | --- | --- | --- |
    | role | combobox | Prefer a native `<select>` or a button with `role="combobox"`. |
    | aria-labelledby | label-id | Associate the visible label, don't duplicate via `aria-label`. |
    | aria-expanded | false | Collapsed; popup not shown. |
    | aria-haspopup | listbox | Opens a listbox of options. |
    | aria-describedby | helper-id | Helper text as description. |

    ### Open & error — ARIA

    | Property | Value | Notes |
    | --- | --- | --- |
    | aria-expanded | true | Popup listbox shown. |
    | aria-activedescendant | active-option-id | Virtually focuses an option without moving DOM focus. |
    | option role | option \+ aria-selected | One selected option; `aria-posinset` / `aria-setsize` for "X of N". |
    | aria-invalid | true | Error state, set on the field. |
    | aria-errormessage / aria-describedby | error-id | Error text replaces helper as the description. |

    <Accordion title="iOS (VoiceOver) & Android (TalkBack)">
      **VoiceOver** — closed: "Country, United Kingdom, button, \[helper\]" via `accessibilityValue` \+ `accessibilityHint`; open: trait stays `.isButton`, value becomes "Expanded", focus moves into option rows ("Option, 1 of 6", `.isSelected` on the chosen one); error folds "invalid data" \+ message into value/hint; disabled uses `.notEnabled`. **TalkBack** — role `Role.DropdownList`, label via `mergeDescendants`, `stateDescription` carries the value then "Expanded"; options use `collectionItemInfo` for position; error via `error(message)` \+ polite live region; disabled via `disabled()`. In both, keep the Field at stop 1 across states, and hide the chevron and scrollbar.
    </Accordion>

    ### Disabled — ARIA

    | Property | Value | Notes |
    | --- | --- | --- |
    | aria-disabled | true | Pair with a non-focusable native disabled control where possible. |
    | Do NOT | keep it in the tab order | Remove tabindex or use a native `disabled` control. |
